# v0.6.0 迭代发布说明

## 概述

v0.6.0 版本专注于增强功能和性能优化，实现了并行执行、错误恢复、条件分支和 Web UI 可视化四大核心功能。

## 新增功能

### 1. 并行执行优化

**功能描述**: 
- 节点按拓扑层级分组，同一组内的节点无依赖关系，可并行执行
- 使用 `Promise.all()` 并行执行同组节点，显著提升执行效率

**类型定义** (`src/types.ts`):
```typescript
// 已在 Graph.ts 中实现
getParallelGroups(): string[][]
```

**实现位置**:
- `src/core/Graph.ts:getParallelGroups()` - 计算并行执行组
- `src/core/Executor.ts:execute()` - 并行执行逻辑

**使用示例**:
```json
{
  "nodes": [
    { "id": "start", ... },
    { "id": "task-a", ... },
    { "id": "task-b", ... }
  ],
  "edges": [
    { "from": { "nodeId": "start" }, "to": { "nodeId": "task-a", "input": "config" } },
    { "from": { "nodeId": "start" }, "to": { "nodeId": "task-b", "input": "config" } }
  ]
}
```
`task-a` 和 `task-b` 将并行执行。

### 2. 错误恢复和重试机制

**功能描述**:
- 支持配置节点失败后的重试策略
- 实现指数退避和固定延迟两种重试策略
- 重试信息记录到审计日志

**类型定义** (`src/types.ts`):
```typescript
export interface RetryConfig {
  maxAttempts?: number;      // 默认 3 次
  backoff?: 'fixed' | 'exponential';  // 重试策略
  delayMs?: number;          // 基础延迟 (ms)
}

export interface BaseNodeConfig {
  retry?: RetryConfig;
  // ... 其他配置
}
```

**实现位置**:
- `src/core/Executor.ts:executeNode()` - 重试逻辑

**使用示例**:
```json
{
  "id": "flaky-node",
  "type": "bash",
  "config": {
    "retry": {
      "maxAttempts": 3,
      "backoff": "exponential",
      "delayMs": 1000
    }
  }
}
```

### 3. 条件分支支持

**功能描述**:
- 边可配置条件表达式，基于上游节点输出决定执行行为
- 支持五种行为：动态路由、跳过边、跳过节点、停止工作流、抛出错误
- 支持多分支动态路由（类似 switch/case）

**类型定义** (`src/types.ts`):
```typescript
export interface EdgeDefinition {
  condition?: {
    expression: string;      // 主条件表达式
    onFalse?: 'skip' | 'skip-node' | 'stop' | 'error';
    branches?: Array<{
      expression: string;     // 分支条件
      to: { nodeId: string; input: string };  // 路由目标
    }>;
  };
  // ... 其他字段
}
```

**行为说明**:

| `onFalse` 值 | 行为 |
|-------------|------|
| `skip` | 跳过这条边（默认） |
| `skip-node` | 跳过整个目标节点 |
| `stop` | 停止整个工作流 |
| `error` | 抛出错误 |

**实现位置**:
- `src/core/Executor.ts:collectInputs()` - 收集输入时评估条件
- `src/core/Executor.ts:evaluateEdgeCondition()` - 条件求值（支持动态路由）

**使用示例 - 动态路由**:
```json
{
  "edges": [
    {
      "id": "route-edge",
      "from": { "nodeId": "start" },
      "to": { "nodeId": "default", "input": "config" },
      "condition": {
        "expression": "outputs['start']?.type !== 'unknown'",
        "branches": [
          {
            "expression": "outputs['start']?.type === 'A'",
            "to": { "nodeId": "handler-a", "input": "config" }
          },
          {
            "expression": "outputs['start']?.type === 'B'",
            "to": { "nodeId": "handler-b", "input": "config" }
          }
        ]
      }
    }
  ]
}
```

**使用示例 - 停止工作流**:
```json
{
  "edges": [
    {
      "condition": {
        "expression": "outputs['health-check']?.allHealthy === true",
        "onFalse": "stop"
      }
    }
  ]
}
```

## 类型系统变更

### 新增类型

```typescript
// RetryConfig - 重试配置
export interface RetryConfig {
  maxAttempts?: number;
  backoff?: 'fixed' | 'exponential';
  delayMs?: number;
}

// ConditionConfig - 条件配置
export interface ConditionConfig {
  expression: string;
  onSkip?: 'pass-empty' | 'error';
}
```

### 修改类型

```typescript
// BaseNodeConfig - 添加 retry 和 condition
export interface BaseNodeConfig {
  retry?: RetryConfig;
  condition?: ConditionConfig;
  // ... 原有字段
}

// EdgeDefinition - 添加 condition
export interface EdgeDefinition {
  condition?: {
    expression: string;
    onSkip?: 'skip-edge' | 'skip-node' | 'error';
  };
  // ... 原有字段
}

// AuditEntry - 添加 retry 字段和 phase
export interface AuditEntry {
  phase: 'start' | 'validate' | 'execute' | 'complete' | 'error' | 'retry';
  retry?: {
    attempt: number;
    maxAttempts: number;
    delay: number;
  };
  // ... 原有字段
}
```

## 向后兼容性

所有变更都是向后兼容的：
- `retry` 和 `condition` 配置是可选的
- 现有工作流无需修改即可运行
- 并行执行对现有工作流透明

---

## 已知限制

1. **并行执行**: 同一组内的节点并行执行，但组间仍是串行
2. **条件表达式**: 目前使用 `new Function()` 求值，仅支持简单的 JavaScript 表达式
3. **Web UI**: 轮询间隔固定为 1 秒，不支持 WebSocket 实时更新
