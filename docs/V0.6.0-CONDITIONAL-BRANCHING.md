# v0.6.0 条件分支功能测试文档

## 功能概述

v0.6.0 的条件边功能已重新设计，支持以下行为：

| 行为 | 说明 |
|------|------|
| **动态路由** | 根据条件表达式结果路由到不同的下游节点 |
| **跳过** | 条件不满足时跳过该边 |
| **跳过节点** | 条件不满足时跳过整个目标节点 |
| **停止工作流** | 条件不满足时停止整个工作流执行 |
| **错误** | 条件不满足时抛出错误 |

## 类型定义

```typescript
interface EdgeDefinition {
  condition?: {
    expression: string;      // 主条件表达式
    onFalse?: 'skip' | 'skip-node' | 'stop' | 'error';
    branches?: Array<{
      expression: string;
      to: { nodeId: string; input: string };
    }>;
  };
}
```

### 字段说明

- **`expression`**: 主条件表达式，基于 `outputs` 对象评估
- **`onFalse`**: 条件为假时的行为
  - `'skip'`: 跳过这条边（默认）
  - `'skip-node'`: 跳过目标节点
  - `'stop'`: 停止整个工作流
  - `'error'`: 抛出错误
- **`branches`**: 动态路由配置，类似 switch/case

## 测试 1: 动态路由

**文件**: `examples/v0.6.0-conditional-test.json`

### 工作流结构

```
                    ┌──────────────┐
                    │    start     │
                    │  status=mode │
                    └──────┬───────┘
                           │
              ┌────────────┴────────────┐
              │                          │
              ▼                          ▼
     ┌─────────────────┐       ┌─────────────────┐
     │  success-path   │       │  failure-path   │
     │  (条件：success)│       │  (条件：failure)│
     └────────┬────────┘       └────────┬────────┘
              │                          │
              └────────────┬────────────┘
                           │
                           ▼
                    ┌──────────────┐
                    │   cleanup    │
                    └──────┬───────┘
                           │
                           ▼
                    ┌──────────────┐
                    │    final     │
                    └──────────────┘
```

### 条件边配置

```json
{
  "id": "edge-start-branch",
  "from": { "nodeId": "start" },
  "to": { "nodeId": "success-path", "input": "config" },
  "condition": {
    "expression": "outputs['start']?.status === 'success'",
    "onFalse": "skip",
    "branches": [
      {
        "expression": "outputs['start']?.status === 'success'",
        "to": { "nodeId": "success-path", "input": "config" }
      },
      {
        "expression": "outputs['start']?.status === 'failure'",
        "to": { "nodeId": "failure-path", "input": "config" }
      }
    ]
  }
}
```

### 运行测试

```bash
# 成功路径测试
node dist/src/cli.js run examples/v0.6.0-conditional-test.json \
  -o /tmp/orc-cond/output -w /tmp/orc-cond/workspace --audit /tmp/orc-cond/audit
```

### 预期结果

```
start → success-path → cleanup → final
```

`failure-path` 因为条件不匹配而被跳过（但在此配置中仍会执行，因为有另一条边连接到它）

## 测试 2: 停止工作流

### 示例配置

```json
{
  "id": "edge-critical",
  "from": { "nodeId": "health-check" },
  "to": { "nodeId": "process-data", "input": "config" },
  "condition": {
    "expression": "outputs['health-check']?.status === 'healthy'",
    "onFalse": "stop"
  }
}
```

### 行为说明

当 `health-check` 返回 `status !== 'healthy'` 时：
1. 工作流立即停止
2. 后续节点不再执行
3. 抛出错误：`Workflow stopped at edge edge-critical: condition not met`

## 测试 3: 跳过节点

### 示例配置

```json
{
  "id": "edge-optional",
  "from": { "nodeId": "check-cache" },
  "to": { "nodeId": "fetch-data", "input": "config" },
  "condition": {
    "expression": "outputs['check-cache']?.hit === false",
    "onFalse": "skip-node"
  }
}
```

### 行为说明

当缓存命中（`hit === true`）时：
1. `fetch-data` 节点被跳过
2. 工作流继续执行后续节点
3. `fetch-data` 的输出为 `undefined`

## 条件表达式语法

### 访问节点输出

```javascript
// 访问节点输出
outputs['node-id']

// 访问嵌套属性
outputs['node-id']?.data?.items

// 数组访问
outputs['node-id']?.results[0]
```

### 比较运算

```javascript
// 相等比较
outputs['node-a']?.status === 'success'

// 数值比较
outputs['node-b']?.count > 10

// 包含检查
outputs['node-c']?.tags?.includes('critical')
```

### 逻辑运算

```javascript
// 与运算
outputs['node-a']?.valid && outputs['node-b']?.valid

// 或运算
outputs['node-a']?.retry || outputs['node-b']?.fallback

// 非运算
!outputs['node-a']?.error
```

### 复杂表达式

```javascript
// 三元运算
outputs['node-a']?.type === 'premium' ? true : outputs['node-b']?.allowed

// 可选链 + 默认值
(outputs['node-a']?.score ?? 0) > 50

// 数组方法
outputs['node-a']?.items?.filter(x => x.valid)?.length > 0
```

## 使用场景

### 1. 环境分支

```json
{
  "condition": {
    "expression": "outputs['init']?.environment === 'production'",
    "branches": [
      {
        "expression": "outputs['init']?.environment === 'production'",
        "to": { "nodeId": "deploy-prod", "input": "config" }
      },
      {
        "expression": "outputs['init']?.environment === 'staging'",
        "to": { "nodeId": "deploy-staging", "input": "config" }
      },
      {
        "expression": "outputs['init']?.environment === 'dev'",
        "to": { "nodeId": "deploy-dev", "input": "config" }
      }
    ]
  }
}
```

### 2. 错误处理路径

```json
{
  "condition": {
    "expression": "outputs['api-call']?.success === true",
    "onFalse": "skip",
    "branches": [
      {
        "expression": "outputs['api-call']?.success === true",
        "to": { "nodeId": "process-response", "input": "data" }
      },
      {
        "expression": "outputs['api-call']?.retryable === true",
        "to": { "nodeId": "retry-call", "input": "config" }
      }
    ]
  }
}
```

### 3. 健康检查门禁

```json
{
  "condition": {
    "expression": "outputs['health-check']?.allHealthy === true",
    "onFalse": "stop"
  }
}
```

## 与旧版本的对比

| 功能 | 旧版本 (onSkip) | 新版本 (onFalse + branches) |
|------|----------------|---------------------------|
| 跳过边 | `onSkip: 'skip-edge'` | `onFalse: 'skip'` |
| 跳过节点 | `onSkip: 'skip-node'` | `onFalse: 'skip-node'` |
| 抛出错误 | `onSkip: 'error'` | `onFalse: 'error'` |
| 停止工作流 | ❌ 不支持 | ✅ `onFalse: 'stop'` |
| 动态路由 | ❌ 不支持 | ✅ `branches: [...]` |

## 注意事项

1. **表达式安全**: 使用可选链 `?.` 避免访问不存在的属性
2. **性能考虑**: 条件表达式使用 `new Function()` 执行，避免复杂计算
3. **循环依赖**: 条件表达式只能访问已执行节点的输出
4. **分支互斥**: branches 按顺序评估，第一个匹配的分支生效
5. **默认行为**: `onFalse` 未配置时默认为 `'skip'`
