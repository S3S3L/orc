# v0.6.0 功能测试文档

## 测试概述

本测试工作流用于验证 ORC v0.6.0 版本新增的三大核心功能：
1. **并行执行** - 无依赖关系的节点并行执行
2. **错误恢复（重试机制）** - 节点失败后自动重试
3. **条件分支** - 基于条件表达式动态控制执行流

## 执行流程说明

### 阶段 1: 启动节点
```
start 节点执行
├── 输出测试配置数据
├── 配置了重试机制（最多 2 次，固定延迟 500ms）
└── 输出：{ timestamp, mode }
```

### 阶段 2: 并行执行（验证并行功能）
```
并行组执行：
├── parallel-a  ←─┐
└── parallel-b  ←─┤ start 输出
                  │
这两个节点无依赖关系，将同时执行
```

### 阶段 3: 条件分支（验证条件功能）
```
conditional-task 节点：
├── 检查边条件：outputs['parallel-a']?.result === 'completed'
├── 条件为真 → 执行节点
└── 条件为假 → 跳过边（skip）
```

### 阶段 4: 汇总（验证多对一输入）
```
final 节点：
├── 收集 parallel-a 的输出（resultA 输入）
└── 收集 parallel-b 的输出（resultB 输入）
```

## 验证的功能点

### 1. 并行执行（Parallel Execution）

**验证点**: `parallel-a` 和 `parallel-b` 同时执行

**工作流配置**:
```json
"edges": [
  { "from": { "nodeId": "start" }, "to": { "nodeId": "parallel-a", "input": "config" } },
  { "from": { "nodeId": "start" }, "to": { "nodeId": "parallel-b", "input": "config" } }
]
```

**预期行为**:
- 两个节点在同一并行组中
- 使用 `Promise.all()` 同时执行
- 执行时间约为单个节点执行时间（而非两者之和）

**验证方法**:
查看审计日志中的执行时间戳：
```bash
cat /tmp/orc-v0.6/audit/parallel-a-*.json | jq '.execution.duration'
cat /tmp/orc-v0.6/audit/parallel-b-*.json | jq '.execution.duration'
```

---

### 2. 错误恢复/重试机制（Retry Mechanism）

**验证点**: `start` 节点配置了重试策略

**工作流配置**:
```json
"config": {
  "retry": {
    "maxAttempts": 2,
    "backoff": "fixed",
    "delayMs": 500
  }
}
```

**预期行为**:
- 如果节点执行失败，自动重试最多 2 次
- 使用固定延迟 500ms
- 重试信息记录到审计日志

**审计日志字段**:
```json
{
  "phase": "retry",
  "retry": {
    "attempt": 1,
    "maxAttempts": 2,
    "delay": 500
  }
}
```

---

### 3. 条件分支（Conditional Branching）

**验证点**: `parallel-a` → `conditional-task` 的边配置了条件

**工作流配置**:
```json
"edges": [
  {
    "id": "edge-a-to-conditional",
    "from": { "nodeId": "parallel-a" },
    "to": { "nodeId": "conditional-task", "input": "dataFromA" },
    "condition": {
      "expression": "outputs['parallel-a']?.result === 'completed'",
      "onFalse": "skip"
    }
  }
]
```

**条件表达式语法**:
- `outputs['node-id']` - 访问节点输出
- `?.` - 可选链操作符
- 支持所有 JavaScript 比较和逻辑运算符

**`onFalse` 选项**:
| 值 | 行为 |
|---|---|
| `skip` | 跳过这条边 |
| `skip-node` | 跳过整个节点 |
| `stop` | 停止工作流 |
| `error` | 抛出错误 |

**动态路由示例**:
```json
"condition": {
  "expression": "outputs['node-a']?.type !== 'unknown'",
  "branches": [
    {
      "expression": "outputs['node-a']?.type === 'A'",
      "to": { "nodeId": "handler-a", "input": "config" }
    },
    {
      "expression": "outputs['node-a']?.type === 'B'",
      "to": { "nodeId": "handler-b", "input": "config" }
    }
  ]
}
```

**onSkip 选项**:
| 值 | 行为 |
|---|---|
| `skip-edge` | 跳过这条边，不传递该输入 |
| `skip-node` | 跳过整个节点执行 |
| `error` | 抛出错误 |

**预期行为**:
- 如果 `parallel-a` 的 `result === 'completed'`，条件为真，执行 `conditional-task`
- 否则，跳过该边（`conditional-task` 可能因缺少输入而失败或跳过）

---

### 4. 多对一输入（Many-to-One）

**验证点**: `final` 节点接收两个上游节点的输出

**工作流配置**:
```json
"edges": [
  { "from": { "nodeId": "parallel-a" }, "to": { "nodeId": "final", "input": "resultA" } },
  { "from": { "nodeId": "parallel-b" }, "to": { "nodeId": "final", "input": "resultB" } }
]
```

**预期行为**:
- `final` 节点的输入：
  ```json
  {
    "resultA": { "task": "A", "result": "completed" },
    "resultB": { "task": "B", "result": "completed" }
  }
  ```

## 故障排查

### 问题：并行节点串行执行
**原因**: 可能存在隐式依赖
**解决**: 检查边配置，确保节点无共同输入依赖

### 问题：重试不生效
**原因**: 脚本退出码为 0 但输出无效
**解决**: 确保失败时使用非零退出码

### 问题：条件分支不执行
**原因**: 条件表达式语法错误或引用不存在的节点
**解决**: 检查 `outputs['node-id']` 中的节点 ID 是否正确
